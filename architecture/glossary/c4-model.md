# C4 model / altitude

**Definition:** a standard way to describe software at four zoom levels — **System → Container →
Component → Code** — so a diagram or note is explicit about *how far zoomed in* it is.

**Answers:** "at what level of detail am I reasoning right now?"

**First encountered in:** [[overview]]

"Altitude" borrows this vocabulary. A concern can be loud at one level and silent at another, so
the vault names its altitude: [[project]]/[[overview]] are **System**, future `proposals/` are
per-feature, future `components/` are **Component**.
